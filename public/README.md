# カフェ空席管理システム (Cafe Seat Manager)

Firebaseを活用し、店側とお客様側をリアルタイムでつなぐ座席管理Webアプリです。
単なる機能実装だけでなく、実務での利用を想定したUX（ユーザー体験）とデータの整合性にこだわって開発しています。

---

## 🛠 実装済みの主要機能

### 1. リアルタイム・フル同期
- **Firestore (onSnapshot)** を採用。
- 店側がステータスを変更した瞬間に、お客様側の画面へ**リロードなしで即座に**反映される双方向同期を実現。

### 2. データの整合性保護（バリデーション）
- **二重登録防止**: `getDoc` と `exists()` を活用し、同一名称の座席重複をブロック。
- **予約矛盾の解消**: 客側画面で「空席」以外の席に対し、予約ボタンを自動で無効化（`disabled`）。不正な上書きを防止。

### 3. オペレーションに最適化したUX
- **誤操作防止**: ステータス変更・削除時に `confirm` ダイアログを表示。
- **マルチデバイス対応**: Firebase Hostingにより、スマホやタブレットからの操作に完全対応。

## 📈 学習の記録と課題解決 (Troubleshooting)

### 🔴 5/8：世界公開（デプロイ）の達成
- **事象**: ローカル環境（localhost）以外からアプリにアクセスできなかった。
- **解決策**: **Firebase Hosting** を導入。CLIによるプロジェクト初期化から `firebase deploy` までのワークフローを習得。
- **学び**: 公開用ディレクトリ（`public`）の概念を理解し、本番環境を見据えたファイル管理を実現。

### 🔴 課題：客側が「利用中」の席を予約できてしまう
- **解決策**: `render` 関数内でステータスを条件分岐させ、`available` 以外はボタンを無効化。システム側で「不正な操作ができない状態」を構築。

### 🔴 課題：修正コードが反映されない「キャッシュ」問題
- **解決策**: 強制リロードおよびLive Serverの再起動。環境要因の切り分け方法を習得。
